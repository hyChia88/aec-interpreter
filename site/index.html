The words, once

A 20-second glossary

BIM — Building Information Model: the digital 3D + data model of a building.
IFC — the open file standard for BIM (vendor-neutral, ISO).
GUID — the globally unique ID each element carries in the model.
Neuro-symbolic — pair a neural network (perception) with deterministic symbolic logic (the executor).
VLM — Vision-Language Model: reads an image + text together.
Spatial address — a relational key ("3rd of 5 windows on this wall") that pins one element among look-alikes.
Oracle — an "if perception were perfect" upper bound; isolates where the real bottleneck is.
Knowledge graph — the model stored as nodes (elements) + typed edges (relations).
The problem

The physical–digital traceability gap

AEC (Architecture, Engineering, Construction) projects standardise on OpenBIM/IFC, yet day-to-day coordination still runs on manual interpretive labour: people translate unstructured evidence — photos, chat messages, floorplan markups — into model-linked, schema-compliant records. The result is information loss across handoffs, ambiguous evidence-to-element links, and days of lag that decay into lost institutional knowledge.

What it does

One question: which element is this?

Given a site photo, a short natural-language note, and (optionally) a marked floorplan, the system returns a ranked shortlist of candidate GUIDs — the exact parts in the BIM model. The hard part is not finding candidates (the right one is in the pool 100% of the time) but discriminating among visually identical siblings: a façade is a row of indistinguishable windows. The signal that separates them is not in their pixels — it is in the building's relational structure, which is exactly what a black-box image matcher fails to recover (it plateaus at 6.7% right-first, with the answer already in the pool).

The system

A neuro-symbolic interpreter middleware

Five stages turn messy evidence into a verifiable answer: multimodal input → neural extraction (a fine-tuned VLM emits a typed Constraints record) → symbolic retrieval (a deterministic, priority-ordered traversal over an IFC knowledge graph) → Graph-RAG reranking → ranked GUIDs. The design rationale is reliability over flexibility:

  • The VLM never writes a database query. It emits a structured {storey, ifc_class, spatial_relations:[…]} contract that fills deterministic templates — so the system can only ever return GUIDs that exist in the model. Zero hallucinated IDs, by construction.
  • The graph is built from the IFC schema itself, not from noisy document extraction — topology is computed, not guessed.
Under the hood

Key modules

IFC Parse & Enrich Engine. Converts raw IFC into a query-ready knowledge graph, enriching topology beyond native containment with retrieval-oriented edges: NEXT_TO, CONNECTS_TO, ADJACENT_TO, ON_STOREY.  IfcOpenShell · Neo4j
Symbolic Planner. A recall-preserving p0∪p1 union planner combines attribute filtering with spatial-topology ranking. Templates are deterministic — same input, same query, same result.  Cypher · priority cascade
Synthetic Data Pipeline. Skeleton-first generation fixes ground truth before adding stochastic visual evidence: IFC mining → Blender renders with hard negatives → LLM text → LLM-as-judge filter → 990 cases, no real labels.  Blender · Gemini · LLM-judge
Deterministic Visual Specialists. OpenCV handles counting and ordinal slot position; ResNet classifies size bands — discriminative cues for sub-tasks VLMs do unreliably.  OpenCV · ResNet

The neural extractor is a fine-tuned Qwen2.5-VL (LoRA) that reaches 82% direction accuracy where zero-shot Gemini scores 0%.

Going deeper — the enhancement

From "the architecture is right" to "the address is recoverable"

The thesis proved the architecture sound: with perfect extraction the symbolic layer retains the correct element in 100% of cases and compresses the candidate pool toward a single answer — so the operational ceiling is bound by perception, not graph logic. The enhancement asks the next two questions and answers them with measurements.

① Representation

What is the minimal sufficient address?

It is type-conditional. A coarse prefix (storey + class) is necessary but saturated — it cannot separate same-class siblings. The discriminator is class-specific: a position-slot (i of M) for an opening, a connectivity fingerprint for a wall.

oracle right-first 4.9% → 78.5%
② Mechanism

How to use a noisy address without losing recall?

Hard filtering deletes the answer; so the address is a soft prior in a recall-fixed pool. One real detector closes most of the gap, its confidence passes a calibration gate, and — since reweighting is a no-op — its payoff is knowing when to abstain.

realized 6.6% → 67.6% · defer → 80.6%
③ Architecture

Where should relational context live?

A depth law: deeper context is more unique but its recovery from an image collapses with distance. So discrimination saturates at one hop — compile the relation into the element, don't chase deep chains.

confusable set 13 → 8 at 1 hop

Worked examples

It answers when sure — and defers when not

Each card traces one case top-to-bottom: ① raw input → ② expected (ground truth) → ③ how the detector reads the plan → ④ predicted address + decision. The predicted spatial address is drawn as a graph (the value on the target node), mirroring the ground-truth graph above it.

ANSWER  Predicted "slot 2 of 17" — ✓ correct; confidence above threshold, so the system commits to the GUID.
DEFER  Predicted "1 of 10" but truth is "8 of 10" (wrong); confidence 0.05 is below threshold, so the system abstains and returns candidates rather than a confident mistake — the triage value proposition.
