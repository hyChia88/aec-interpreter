An Interpreter Layer for AEC Grounding a construction-site photo and a field note to the exact element in the building's digital model — hallucination-free and traceable.

A building's digital model — its BIM — stores thousands of look-alike parts: hundreds of near-identical windows and doors, each with a globally-unique ID (GUID). Point a phone at one, add a short note, and this AI "interpreter" tells you which modelled element it is — automatically, using only the BIM model itself, with no history of labelled site photos to learn from, and — crucially — it refuses to guess when it is unsure.

The problem

The physical–digital traceability gap

Construction and facilities teams standardise on OpenBIM/IFC, yet day-to-day coordination still runs on manual interpretive labour: people translate unstructured evidence — photos, chat messages, floorplan markups — into model-linked records by hand. The final step is choosing one element from a retrieved shortlist that already contains the answer (median 76 candidates, ground truth present 100% of the time). So the difficulty is not retrieval but discriminating among visually identical siblings — a façade is a row of indistinguishable windows. What separates two windows is not in their pixels; it is in the building's relational structure, which a black-box image matcher must reconstruct internally and unreliably — and it plateaus at just 6.7% right-first, with the answer already in the pool. Making that relational signal explicit is the contribution.

The system

One neuro-symbolic interpreter — perception in, a verifiable element out

Neuro-symbolic means a neural network does the perceiving and a deterministic symbolic engine does the deciding. Evidence flows through one pipeline:

① INPUT

Evidence

site photo + field note + (optional) marked floorplan

→
② PERCEPTION

Neural extraction

a fine-tuned VLM + deterministic visual specialists read the evidence into a typed, structured description

→
③ SYMBOLIC

Graph retrieval

a deterministic planner queries the IFC knowledge graph; a Graph-RAG reranker orders the candidates

→
④ CONTROL

Route & decide

an orchestration layer routes by confidence and answers — or defers to a human

→
⑤ OUTPUT

Ranked GUIDs

linked back to the BIM model

The load-bearing design choice is reliability over flexibility: the neural network never writes a database query. It only emits a structured description; deterministic templates turn that into the query. So the system can only ever return GUIDs that exist in the model — zero hallucinated IDs, by construction — and every error is traceable to a specific stage (type, floor, or ranking).

Fine-tuned VLM

The perception engine

A LoRA-fine-tuned Qwen2.5-VL reads the photo, note, and floorplan and emits a structured contract — {storey, ifc_class, spatial_relations:[{predicate, object_type…}]}. It describes, it never queries. Fine-tuning on the synthetic corpus lifts spatial-direction accuracy to 82% where zero-shot Gemini scores 0%.

Qwen2.5-VL · LoRA · typed Constraints JSON
Deterministic visual specialists. OpenCV counts openings and reads ordinal position; ResNet classifies size bands — the sub-tasks a VLM does unreliably. Each emits its own confidence, and this is the seed of the spatial-interpretation work below.  OpenCV · ResNet

③ Symbolic backend — the deterministic engineAn IFC-native knowledge graph + a deterministic planner

IFC Parse & Enrich Engine. Converts the raw IFC into a knowledge graph — nodes are elements, edges are relations — enriched beyond native containment with retrieval-oriented topology: NEXT_TO, CONNECTS_TO, ADJACENT_TO, ON_STOREY.  IfcOpenShell · Neo4j
Symbolic Planner + Graph-RAG. A recall-preserving p0∪p1 union planner combines attribute filtering with spatial-topology ranking; templates are deterministic — same input, same query, same result.  Cypher · priority cascade

④ Control — adaptive routingAn orchestration layer that knows when to defer

Orchestration & routing

Confidence-driven control

Every extracted field carries a {value, confidence, source} record. An orchestration layer routes on that confidence — which deterministic specialist to trust, which constraint to apply softly, and when to defer to a human instead of guessing. The policy is studied as an ablation, from a static calibrated threshold → a learned router → an LLM-agent orchestrator (same backbone), scored on accuracy, latency, and repeatability.

static → learned router → agent · the adaptivity arm
Trained with zero real labels. Skeleton-first generation fixes ground truth before adding stochastic visual evidence: IFC mining → Blender renders with hard negatives → LLM text → LLM-as-judge filter → 990 cases, no human annotation.  Blender · Gemini · LLM-judge
The innovation

Reading space from a flat image — the spatial-interpretation layer

The thesis proved the architecture sound: with perfect perception the symbolic engine keeps the correct element 100% of the time and compresses the shortlist toward a single answer — so the bottleneck is perception, not graph logic. The core advance deepens the deterministic visual specialists into a principled spatial-interpretation layer: instead of ad-hoc cues, the system recovers a spatial address — a relational key read from plain 2D images and floorplans, computable from the BIM model and recoverable from the evidence. Three questions, answered with measurements:

① Representation

What is the minimal address?

It is type-conditional: a coarse prefix (storey + class) is necessary but saturated; the discriminator is class-specific — a position-slot (i of M) for an opening, a connectivity fingerprint for a wall.

right-first 4.9% → 78.5% (oracle)
② Mechanism

How to use a noisy address?

Hard filtering deletes the answer, so the address is a soft prior in a recall-fixed pool. One real detector closes most of the gap; its confidence passes a calibration gate, and its payoff is knowing when to abstain — not blind reweighting.

realized 6.6% → 67.6% · defer → 80.6%
③ Architecture

How deep should context go?

A depth law: deeper relations are more unique but their recovery from an image collapses with distance, so discrimination saturates at one hop — compile the relation into the element rather than chase deep chains.

confusable set 13 → 8 at one hop

Worked examples

It answers when sure — and defers when not

Each card traces one case top-to-bottom: ① raw input → ② expected (ground truth) → ③ how the detector reads the plan → ④ predicted address + decision. The predicted address is drawn as a graph (the value on the target node), mirroring the ground-truth graph above it.

ANSWER  Predicted "slot 2 of 17" — ✓ correct; confidence above threshold, so the system commits to the GUID.
DEFER  Predicted "1 of 10" but truth is "8 of 10" (wrong); confidence 0.05 is below threshold, so the system abstains and returns candidates rather than a confident mistake — the triage value proposition.
